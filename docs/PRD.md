## 1. 产品概述

### 1.1 产品定位

**AIBILL** Android App 是一款 AI 驱动的智能记账应用，作为 AIBILL 生态的原生移动端，复用已有的后端 API 服务，为用户提供：

- **极速记账体验**：自然语言输入 + AI 自动解析，3 步内完成记账
- **原生独有能力**：通知监听自动记账、语音输入、桌面小组件
- **离线可用**：无网络时手动记账不中断，联网后自动同步

### 1.2 产品愿景

让记账像发微信一样简单 —— 说一句话、收一条通知，账就记好了。

### 1.3 核心体验指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 打开→入账 | ≤ 3 步 | 从启动 App 到完成一笔记账 |
| AI 记账耗时 | < 5 秒 | 用户输入到确认完成 |
| 离线可用 | 100% | 手动记账不依赖网络 |
| 冷启动时间 | < 2 秒 | 首屏可交互 |
| 崩溃率 | < 0.1% | 线上稳定性 |

## 2. 项目背景

### 2.1 现有系统

AIBILL 目前已有完整的产品形态：

- **Web 端**：Vue3 PWA 应用，支持 AI 记账、统计分析、预算管理
- **后端 API**：Node.js + Fastify，提供 RESTful 接口
- **AI 能力**：自然语言解析记账、智能对话查账

### 2.2 为什么需要原生 App

| 痛点 | Web 端局限 | App 解决方案 |
|------|------------|--------------|
| 记账不及时 | 需要主动打开浏览器 | 通知监听自动识别支付 |
| 输入效率低 | 键盘打字 | 语音输入 + 快捷短语 |
| 桌面入口深 | PWA 体验不够原生 | 桌面小组件一键记账 |
| 离线不可用 | PWA 离线支持有限 | Room 本地存储 + 同步队列 |
| 推送提醒缺失 | 无原生推送 | 预算超支/定时提醒 |

### 2.3 技术约束

- 复用现有后端 API，不新增后端开发
- API Base URL: `http(s)://<server-ip>:3000/api`
- 认证方式: JWT Bearer Token (30 天有效期)
- 金额单位: 分 (整数)，UI 展示时 ÷ 100
- 数据隔离: 每个用户只能访问自己的数据

### 2.4 首次使用流程

**首次启动流程**：
```
启动 App
  → 服务器地址配置（输入自己的服务端 IP/域名）
  → 登录（已有账号直接登录）
  → 进入首页，开始使用
```

**设计要点**：
- 服务器地址持久化存储，后续启动自动连接
- 支持 HTTP/HTTPS 两种协议
- 地址输入支持自动补全 `http://` 前缀和 `:3000/api` 后缀
- 设置页可随时修改服务器地址

## 3. 用户画像

### 3.1 目标用户

本产品为个人自用项目，核心用户就是自己：
- 有自部署服务器，注重数据隐私和自主可控
- 日常高频使用微信/支付宝支付，希望自动记录
- 有记账意愿但怕麻烦，需要极简的输入方式

### 3.2 核心使用场景

1. **即时记账**：午饭后掏出手机，说"午饭32"，AI 自动分类确认入账
2. **被动记账**：支付完成收到通知，App 自动识别并生成待确认账单
3. **查账分析**：月底想看花了多少，问 AI "这个月餐饮花了多少"
4. **离线记账**：地铁/飞机无网络时手动记一笔，联网后自动同步

## 4. 核心功能模块

### 4.1 AI 智能记账 (P0 - 核心)

**功能描述**：用户用自然语言输入，AI 自动解析为结构化交易数据。

**交互流程**：
```
用户输入 "午饭32，打车15"
    → 调用 POST /api/ai/parse
    → 返回解析结果（类型、金额、分类、描述）
    → 展示确认卡片（支持逐条修改）
    → 用户确认 → 调用 POST /api/transactions 入库
```

**输入方式**：
- 文本输入：首页输入框直接打字
- 语音输入：长按录音按钮，系统语音识别转文字后走 AI 解析
- 快捷短语：常用记账短语一键填入（如"早餐"、"地铁"）

**异常处理**：
- AI 解析失败 → 提示用户切换手动记账
- 网络超时 → 本地暂存，标记待同步
- 解析结果不准确 → 确认卡片支持修改每个字段

### 4.2 手动记账 (P0 - 核心)

**功能描述**：传统表单方式录入交易，支持离线使用。

**字段**：
| 字段 | 必填 | 说明 |
|------|------|------|
| 类型 | ✅ | 支出/收入/转账 |
| 金额 | ✅ | 数字键盘输入，UI 显示元 |
| 分类 | ✅ | 图标网格选择（转账时无需） |
| 账户 | ❌ | 默认使用用户设置的默认账户 |
| 目标账户 | ❌ | 仅转账时必填 |
| 描述 | ❌ | 备注说明 |
| 日期 | ✅ | 默认今天，可修改 |
| 时间 | ❌ | 可选 |
| 标签 | ❌ | 多标签 |

### 4.3 通知监听自动记账 (P1 - App 独有)

**功能描述**：监听系统通知栏，自动识别支付类通知并解析为交易。

**实现方式**：
- 使用 `NotificationListenerService` 监听系统通知
- 正则匹配 + AI 辅助解析通知文本
- 生成待确认交易 → 用户确认后入库

**支持的通知来源**：
| 来源 | 识别方式 | 示例 |
|------|----------|------|
| 微信支付 | 包名 + 正则 | "微信支付成功，付款￥32.00" |
| 支付宝 | 包名 + 正则 | "支付宝付款￥15.00" |
| 银行短信 | 关键词正则 | "尾号1234消费￥500.00" |

**实时弹窗确认**：

当检测到新账单时，立即弹出悬浮确认窗口（Heads-up 风格），用户无需离开当前应用即可完成确认：

```
┌─────────────────────────────────────┐
│  💰 检测到一笔支出                    │
│                                     │
│  微信支付 · ¥32.00                   │
│  沙县小吃                            │
│  分类：餐饮 · 账户：微信              │
│                                     │
│  [忽略]     [修改]     [✓ 确认]      │
└─────────────────────────────────────┘
```

**弹窗交互规则**：
| 场景 | 行为 |
|------|------|
| App 在前台 | 顶部 Snackbar/Dialog 弹窗，不遮挡主要内容 |
| App 在后台 | Heads-up 通知（可展开操作），点击"确认"直接入库 |
| 屏幕锁定 | 锁屏通知展示摘要，解锁后可操作 |
| 用户未操作 | 10 秒后自动收起，进入通知中心待确认列表 |
| 连续多笔 | 队列依次弹出，不堆叠轰炸 |
| 点击"修改" | 跳转到编辑页面，预填解析数据 |
| 点击"确认" | 直接入库（调 API 或存本地待同步） |
| 点击"忽略" | 标记 ignored，不再提醒 |

**通知动作按钮**（App 在后台时）：
- 使用 Android Notification Action Button，用户无需打开 App 就能确认
- 支持 "确认" / "忽略" 两个 Action
- 点击通知正文 → 打开 App 进入编辑页面

**用户控制**：
- 全局开关：设置中开启/关闭
- 应用白名单：选择监听哪些 App 的通知
- 弹窗模式选择：实时弹窗 / 仅通知栏 / 静默入库（高级用户可直接自动确认）
- 免打扰时段：指定时间段内不弹窗，静默存入待确认列表
- 待确认列表：通知中心统一查看，支持批量确认/忽略

### 4.4 流水管理 (P0 - 核心)

**功能描述**：查看、搜索、编辑、删除交易记录。

**特性**：
- 按日分组展示，含日小计
- 搜索：关键词、日期范围、类型、分类、账户多维度筛选
- 编辑：点击进入详情编辑
- 删除：左滑删除（软删除）
- 分页加载：下拉刷新 + 上拉加载更多

### 4.5 统计分析 (P1)

**功能描述**：多维度可视化收支数据。

**视图**：
- **收支摘要**：月收入/支出/结余 + 环比变化
- **趋势图**：日/周/月支出趋势折线图
- **分类饼图**：按分类占比展示支出构成
- **排行列表**：按金额排序的分类/标签排行

### 4.6 AI 对话 (P1)

**功能描述**：与 AI 自然语言对话，查询和分析账务。

**示例**：
- "这个月花了多少？"
- "上个月餐饮和这个月比怎么样？"
- "最近一周最大的一笔支出是什么？"

### 4.7 预算管理 (P1)

**功能描述**：设置月度预算，追踪执行情况。

**特性**：
- 总预算 + 分类预算
- 进度条可视化剩余额度
- 超支推送提醒

### 4.8 桌面小组件 (P2)

| Widget | 功能 | 尺寸 |
|--------|------|------|
| 快速记账 | 点击直接打开 AI 输入 | 2×1 |
| 月度摘要 | 显示本月支出/收入/结余 | 4×2 |

### 4.9 离线暂存与同步 (P0)

**机制**：
```
离线时：
  → 手动记账存入本地 Room 数据库
  → client_id (UUID) 持久化保存
  → 标记 syncStatus = "pending"

联网后：
  → WorkManager 触发同步任务
  → 批量调 POST /api/transactions
  → 利用 client_id 幂等保证不重复
  → 成功后标记 syncStatus = "synced"
  → 失败则标记 "failed" 并支持重试
```

**冲突处理策略**：

| 场景 | 策略 | 说明 |
|------|------|------|
| 新增冲突 | client_id 幂等 | 同一 client_id 重复提交返回已有记录 |
| 编辑冲突 | 最后写入胜出 (Last Write Wins) | 使用 `updatedAt` 时间戳判断，客户端时间较新则覆盖 |
| 删除冲突 | 删除优先 | 任一端删除则最终删除 |
| 离线编辑已同步记录 | 乐观锁 | 带 `version` 字段提交，冲突时提示用户选择 |

**本地缓存策略**：
- 默认缓存最近 7 天的流水数据（离线可查看）
- 分类和账户列表完整缓存（体积小，全量同步）
- 统计数据按月缓存，联网时刷新

### 4.10 深色模式 (P0)

2026 年深色模式是基本需求，v0.1 即支持：
- 跟随系统 / 浅色 / 深色 三种模式
- Compose 动态主题适配
- 所有图表组件适配深色配色

### 4.11 消费智能分类学习 (P1)

**功能描述**：根据用户历史修改行为，自动学习分类偏好。

**机制**：
- 用户手动修改 AI 解析的分类后，本地记录「描述关键词 → 分类」映射
- 下次相同关键词出现时，优先使用本地映射结果
- 本地规则优先于 AI 解析（减少 AI 调用 + 更准确）

示例：用户第一次将"瑞幸"从"餐饮-其他"改为"餐饮-咖啡" → 以后瑞幸消费自动归类为"餐饮-咖啡"

## 5. 页面结构与交互设计

### 5.1 应用导航架构

```
App
├── 启动页 (Splash)
│   └── Token 校验 → 有效则进首页，无效进登录
├── 登录/注册
│   ├── 登录页（用户名 + 密码）
│   └── 注册页（用户名 + 密码 + 邀请码 + 昵称?）
├── 主框架 (BottomNav)
│   ├── Tab1: 首页
│   ├── Tab2: 流水
│   ├── Tab3: 统计
│   ├── Tab4: 我的
│   └── FAB: 快速记账入口
├── 独立页面
│   ├── AI 对话页
│   ├── 手动记账页
│   ├── 交易详情/编辑页
│   ├── 预算管理页
│   ├── 通知中心（待确认交易）
│   └── 设置子页面
└── Widget
    ├── 快速记账 Widget
    └── 月度摘要 Widget
```

### 5.2 各页面详细设计

#### 5.2.1 首页 (Home)

**设计原则**：首页只做一件事 —— 让用户最快速地完成记账。信息精简，避免过载。

**布局结构**（从上到下）：

| 区域 | 内容 | 交互 |
|------|------|------|
| 顶部 Bar | 月份 + 月支出总额 + 通知铃铛（有未确认时红点） | 点击铃铛进通知中心 |
| AI 输入区 | 文本输入框 + 语音按钮 | 核心区域，视觉最突出 |
| 快捷短语 | 横向滚动标签（早餐/午餐/地铁/咖啡...） | 点击直接填入输入框 |
| 确认卡片 | AI 解析结果预览 + **全部确认按钮** | 单条修改/确认，或一键全部确认 |
| 今日流水 | 今天的交易列表（最近 3-5 条） | 点击进详情，底部"查看全部" |

**关键交互**：
- AI 输入框获取焦点时，键盘弹起，快捷短语横向滚动显示
- 语音按钮长按开始录音，松开结束，文字填入输入框
- 多条解析结果时显示"全部确认"按钮 + 展开逐条查看
- 确认卡片支持左右滑动逐条查看，也可一键全部确认
- 下拉刷新同步数据
- 首页**不展示**月度摘要详情卡片（精简，统计页再看详细）

#### 5.2.2 流水页 (Transactions)

**布局结构**：

| 区域 | 内容 | 交互 |
|------|------|------|
| 搜索栏 | 搜索关键词 + 筛选按钮 | 筛选弹出 BottomSheet |
| 筛选条件 | 日期范围/类型/分类/账户 | 多选 chips |
| 流水列表 | 按日分组，含日期标题和日小计 | 上拉加载，下拉刷新 |
| 流水项 | 图标 + 分类 + 描述 + 金额 + 时间 | 点击进详情，左滑删除 |

**关键交互**：
- 搜索实时过滤（防抖 300ms）
- 删除二次确认
- 列表空状态引导记账

#### 5.2.3 统计页 (Statistics)

**布局结构**：

| 区域 | 内容 | 交互 |
|------|------|------|
| 时间选择 | 月份切换（左右箭头） | 切换月份刷新数据 |
| Tab 切换 | 支出 / 收入 | 切换刷新下方内容 |
| 摘要区 | 总额 + 环比百分比 | - |
| 趋势图 | 日支出折线图 | 点击数据点显示具体值 |
| 饼图 | 分类占比环形图 | 点击扇区高亮 + 显示具体 |
| 排行列表 | 分类排行（金额 + 百分比 + 进度条） | 点击进入该分类流水 |

#### 5.2.4 我的页 (Profile)

**布局结构**：

| 区域 | 内容 |
|------|------|
| 用户信息 | 头像 + 昵称 + 账户名 |
| 功能入口 | AI 对话 / 预算管理 / 数据导出 |
| 设置入口 | 账户管理 / 分类管理 / 通知设置 / 通用设置 |
| 其他 | 关于 / 退出登录 |

#### 5.2.5 手动记账页

**布局结构**：

| 区域 | 内容 | 交互 |
|------|------|------|
| 类型切换 | 支出/收入/转账 Tab | 切换影响下方分类列表 |
| 金额输入 | 大字体金额 + 自定义数字键盘 | 支持小数点，自动转分 |
| 分类选择 | 图标网格（3×n） | 点选高亮 |
| 更多字段 | 账户/日期/描述/标签（可收起） | 默认折叠，点展开 |
| 底部按钮 | 保存 | 校验后提交 |

#### 5.2.6 AI 对话页

**布局结构**：
- 顶部：标题 + 新会话按钮
- 中间：对话消息列表（气泡样式）
- 底部：输入框 + 发送按钮

**消息类型**：
- 用户消息（右侧气泡）
- AI 回复（左侧气泡，支持 Markdown 渲染）

#### 5.2.7 通知中心

**功能**：展示所有通知监听自动生成的待确认交易。

| 操作 | 说明 |
|------|------|
| 确认 | 将 pending 交易标记为 confirmed 并同步 |
| 编辑 | 修改解析结果后确认 |
| 忽略 | 丢弃该条记录 |
| 全部确认 | 批量确认所有待处理交易 |

## 6. 技术架构

### 6.1 技术选型

| 层级 | 技术方案 | 说明 |
|------|----------|------|
| 语言 | Kotlin | Android 官方推荐 |
| UI 框架 | Jetpack Compose | 声明式 UI，现代化开发体验 |
| 架构模式 | MVVM + Clean Architecture | 分层清晰，易测试 |
| 网络层 | Retrofit + OkHttp + Moshi | 成熟方案，JSON 序列化 |
| 本地存储 | Room (SQLite) | 离线数据 + 缓存 |
| 轻量存储 | DataStore (Proto) | 用户设置、Token 存储 |
| 依赖注入 | Hilt | Google 官方 DI 方案 |
| 异步处理 | Coroutines + Flow | 响应式数据流 |
| 图片 | Coil | Compose 原生支持 |

- 切换地址时清空本地缓存

## 8.5 AI 调用成本控制策略

> 自部署场景下 AI 调用也有成本（API 费用或自建模型的算力），需要控制频率。

**分层解析策略**：
```
用户输入 / 通知文本
     │
     ▼
[1. 本地规则引擎（优先）]
  - 用户历史分类映射（"瑞幸"→餐饮-咖啡）
  - 正则匹配（通知文本提取金额/商家）
  - 快捷短语直接映射（"午餐"→餐饮 ¥20）
     │
     │ 匹配成功 → 直接生成交易，不调 AI
     │ 匹配失败 ↓
     ▼
[2. AI 解析（兜底）]
  - 调用 POST /api/ai/parse
  - 结果缓存到本地规则库（下次同类输入不再调 AI）
```

**效果预期**：
- 日常 70%+ 的记账可通过本地规则完成（通知正则 + 历史学习）
- AI 调用主要发生在用户新输入和首次使用阶段
- 随着使用时间增长，AI 调用频率逐步降低

## 8.6 自动化体验设计（核心差异化）

> 终极目标：**零操作记账** —— 用户什么都不用做，账就自动记好了。

### 8.6.1 自动化分级体系

根据"置信度"决定是否需要用户确认：

```
┌─────────────────────────────────────────────────────┐
│              自动化置信度模型                          │
├──────────┬──────────┬───────────────────────────────┤
│ 置信度    │ 行为     │ 适用场景                       │
├──────────┼──────────┼───────────────────────────────┤
│ 高 (≥90%)│ 静默入库  │ 历史完全匹配（同商家+同金额）  │
│          │ 不打扰   │ 或用户标记为"信任"的来源        │
├──────────┼──────────┼───────────────────────────────┤
│ 中 (60-90)│ 弹窗确认 │ 金额识别准确但分类有猜测成分   │
│          │ 可一键   │ 新商家、金额异常等              │
├──────────┼──────────┼───────────────────────────────┤
│ 低 (<60%)│ 存入待审 │ 文本模糊、多义解析、格式异常    │
│          │ 不弹窗   │ 等待用户主动处理               │
└──────────┴──────────┴───────────────────────────────┘
```

**置信度计算因素**：
- 商家是否在历史记录中出现过 → +30%
- 金额是否明确提取到 → +30%
- 分类是否与历史一致 → +20%
- 通知来源是否在信任列表 → +10%
- 正则是否完整匹配模板 → +10%

### 8.6.2 规则自学习引擎

不只是"用了 AI 就聪明了"，而是**越用越不需要 AI**：

```
第 1 次：在"瑞幸"消费 ¥28 → AI 解析 → 用户确认（改分类为"咖啡"）
第 2 次：在"瑞幸"消费 ¥32 → 本地匹配"瑞幸"→ 自动分类"咖啡"→ 弹窗确认
第 3 次：在"瑞幸"消费 ¥28 → 本地匹配 → 置信度 95% → 静默入库
```

**学习维度**：

| 维度 | 学习内容 | 示例 |
|------|----------|------|
| 商家→分类 | 描述关键词映射分类 | "瑞幸"→咖啡，"滴滴"→交通 |
| 商家→账户 | 特定商家常用支付方式 | "美团"→微信支付 |
| 时间→分类 | 时间段偏好 | 7-9点→早餐，12-13点→午餐 |
| 金额→分类 | 金额范围猜测 | ¥2-5→交通（地铁），¥15-40→餐饮 |
| 来源→信任度 | 哪些通知来源可信 | 微信支付通知→信任，促销通知→忽略 |

### 8.6.3 智能免确认规则

用户可自定义"哪些情况不需要我确认"：

| 规则类型 | 示例 | 效果 |
|----------|------|------|
| 固定金额免确认 | 地铁每次 ¥3-5 | 直接静默入库 |
| 固定商家免确认 | 瑞幸/美团/滴滴 | 匹配即入库 |
| 小额免确认 | ≤ ¥10 的支出 | 全部静默，月末批量回顾 |
| 固定时段免确认 | 工作日 12:00-13:00 | 午餐时段自动记 |
| 信任来源免确认 | 微信支付成功通知 | 该通知源全部自动 |

**设置页面交互**：
```
自动记账设置
├── 自动化程度：[全手动] ←滑块→ [全自动]
│     保守 ─── 标准 ─── 激进
│   (全部确认)  (智能判断)  (只存不问)
├── 免确认规则
│   ├── + 小额免确认（≤ ¥__）
│   ├── + 固定商家（选择/添加）
│   └── + 自定义规则...
└── 月末回顾提醒（开/关）
    "本月有 47 笔自动入库的账，要检查一下吗？"
```

### 8.6.4 场景化自动触发

不只被动等通知，主动感知场景：

| 触发条件 | 动作 | 说明 |
|----------|------|------|
| 连接公司 WiFi | 预填"午餐"模板 | 12 点弹出"记午餐？" |
| 断开共享单车蓝牙 | 弹出交通记账 | 骑行结束自动提示 |
| 日历事件（发薪日） | 自动记录工资收入 | 每月固定日期触发 |
| 检测到外卖通知 | 等待支付通知关联 | 先看到外卖下单，再匹配付款通知 |
| 夜间时段（22:00+）| 弹出"今日未记账提醒" | 如果当天 0 笔手动记账 |

### 8.6.5 通知链关联识别

高级场景：将多条相关通知关联为一笔完整交易。

```
通知1 (12:05): "美团外卖订单已送达"          → 识别：外卖
通知2 (12:05): "微信支付成功，¥28.5"        → 识别：金额
     ↓ 关联（时间窗口 5 分钟内 + 金额匹配）
最终结果：支出 ¥28.5，分类"餐饮-外卖"，描述"美团外卖"，账户"微信"
```

**关联规则**：
- 同一分钟内多条通知 → 尝试关联
- 先有商品通知 + 后有支付通知 → 合并为一条
- 避免重复记账（外卖确认 + 支付确认 = 只记 1 笔）

### 8.6.6 Tasker/Intent 对外接口

暴露标准 Android Intent 接口，让用户可以用 Tasker 等自动化工具触发记账：

```kotlin
// 外部调用记账 Intent
Intent("com.aibill.ACTION_QUICK_RECORD").apply {
    putExtra("amount", 3200)        // 分
    putExtra("type", "expense")
    putExtra("category", "餐饮")
    putExtra("description", "午饭")
    putExtra("auto_confirm", true)  // 是否跳过确认
}

// 外部调用 AI 解析
Intent("com.aibill.ACTION_AI_PARSE").apply {
    putExtra("input", "午饭32 打车15")
}
```

**适用场景**：
- Tasker 检测到 NFC 标签 → 触发固定记账（贴在公交卡上）
- Tasker 检测到蓝牙断开 → 触发交通记账
- 其他 App 通过 Intent 调用本 App 记账

### 8.6.7 自动化程度渐进策略

用户不需要一开始就配置复杂规则，系统随使用逐步变聪明：

```
Week 1:  全部需要确认，系统在后台学习用户习惯
Week 2:  开始提示"瑞幸消费已出现3次，是否设为免确认？"
Week 3:  高频商家自动免确认，只有新商家/异常金额才弹窗
Week 4+: 80%+ 交易静默入库，用户只处理异常
```

**关键体验**：月末推送一条汇总 "本月共自动记录 XX 笔，为你节省了约 XX 分钟"，让用户感知到自动化的价值。

## 8.6 用户留存与坚持激励

> 核心问题：用户放弃记账的最大原因是"断了就不想续了"。

**防放弃机制**：

| 机制 | 说明 |
|------|------|
| 自动记账兜底 | 通知监听 + 自动确认模式，即使用户懒得操作也有数据 |
| 积压宽容 | 待确认列表堆积时不焦虑提醒，而是提供"一键全部确认" |
| 记账打卡 | 连续记账天数统计，简单的数字激励 |
| 周报推送 | 每周一推送"上周消费摘要"，让用户感知价值 |
| 消费洞察 | "本月奶茶花了 ¥320，比上月多 40%" 这类有趣的发现 |
| 里程碑 | "已记录第 100 笔" "累计管理 ¥50,000" 等成就通知 |

**设计原则**：
- 永远不要让用户觉得"断了就废了" —— 任何时候回来都有自动积累的数据
- 激励要轻量，不要搞排行榜/社交压力那套
- 推送频率克制：每周最多 2 条，用户随时可关

## 附录 A：移动端特有能力补充（行业调研）

> 基于 AutoAccounting（自动记账 4.0）、钱迹、随手记、咔皮记账、小米钱包记账等主流方案调研整理。以下为 Android 原生端相比 Web 端的差异化能力清单。

### A.1 多渠道账单自动捕获

行业做法是**三管齐下**，不只是通知监听：

| 渠道 | 实现方式 | 优先级 | 说明 |
|------|----------|--------|------|
| **应用通知** | NotificationListenerService | P0 | 最通用，无需 Root。微信/支付宝/银行推送 |

**补充到 PRD 的调整**：
- 通知监听保持 P1 不变

### A.7 权限引导（懒人式）

Android 记账 App 涉及多项敏感权限，用户不知道去哪里开、为什么要开。设计原则：**用到时再问 + 一键跳转 + 实时检测**。

#### 权限清单与引导时机

| 权限 | 用途 | 引导时机 | 引导方式 |
|------|------|----------|----------|
| 通知监听 (NotificationListener) | 自动捕获支付通知 | 首次开启"自动记账"开关时 | 引导弹窗 → 一键跳转系统设置 |
| 短信读取 (READ_SMS) | 解析银行扣款短信 | 首次开启"短信记账"开关时 | 运行时权限弹窗 |
| 悬浮窗 (SYSTEM_ALERT_WINDOW) | 实时弹窗确认 | 首次触发自动记账弹窗时 | 引导弹窗 → 跳转设置 |
| 录音 (RECORD_AUDIO) | 语音输入 | 首次点击语音按钮时 | 运行时权限弹窗 |
| 相机 (CAMERA) | 拍照识别小票 | 首次点击拍照按钮时 | 运行时权限弹窗 |
| 定位 (ACCESS_FINE_LOCATION) | 消费地点记录 | 首次开启"记录位置"开关时 | 运行时权限弹窗 |
| 后台运行 | 保证通知监听不被杀 | 开启自动记账后检测到被杀时 | 引导去电池优化白名单 |

#### 引导弹窗设计

```
┌──────────────────────────────────────┐
│  🔔 开启通知监听                      │
│                                      │
│  开启后，App 可以自动识别微信、        │
│  支付宝等支付通知，帮你自动记账。      │
│                                      │
│  ⚠️ 需要到系统设置中手动开启          │
│                                      │
│  ┌────────────────────────────────┐  │
│  │ 1. 点击下方按钮跳转设置         │  │
│  │ 2. 找到「AIBILL」并开启开关     │  │
│  │ 3. 返回 App 自动生效            │  │
│  └────────────────────────────────┘  │
│                                      │
│      [以后再说]     [去开启 →]        │
└──────────────────────────────────────┘
```

#### 实时状态检测与提醒

| 机制 | 说明 |
|------|------|
| **返回自动检测** | 用户从系统设置返回 App 后，自动检测权限状态并刷新 UI |
| **设置页状态展示** | 每项权限用 ✅/❌ 标识当前状态，未开启的显示"去开启"按钮 |
| **功能降级提示** | 权限未授予时，相关功能入口显示提示角标"需要权限" |
| **首页健康检查卡片** | 检测到关键权限缺失时，首页顶部展示提示卡片（可关闭） |
| **省电策略引导** | 检测到 App 被系统杀后台时，引导用户关闭电池优化/加入白名单 |

#### 省电/保活引导（各厂商适配）

针对国产 ROM 的后台限制，自动检测并给出对应引导：

| 品牌 | 引导内容 |
|------|----------|
| 小米 MIUI/HyperOS | 设置 → 应用 → 自启动 + 省电策略"无限制" |
| 华为 EMUI/HarmonyOS | 设置 → 电池 → 启动管理 → 手动管理 |
| OPPO ColorOS | 设置 → 电池 → 自启动管理 |
| vivo OriginOS | 设置 → 电池 → 后台高耗电 |
| 三星 OneUI | 设置 → 电池 → 后台使用限制 → 从不休眠 |

**实现**：通过 `Build.MANUFACTURER` 识别品牌，动态展示对应的图文引导步骤，点击按钮直接 Intent 跳转到对应设置页面。

#### 设计原则

1. **按需申请**：不在首次启动时一股脑要全部权限，用到哪个要哪个
2. **说人话**：解释清楚权限用来干嘛，而不是系统冷冰冰的描述
3. **一键直达**：每个权限引导都带"跳转"按钮，不让用户自己翻设置
4. **永不卡死**：拒绝权限后功能降级但不崩溃，随时可以再去开启
5. **状态可见**：设置页统一展示所有权限状态，一目了然

- 首次启动（服务器地址配置 + 登录）
- 登录/注册（邀请码可选）
- AI 文本记账（含多条"全部确认"）
- 手动记账（计算器键盘 + 连续模式）
- 流水列表（搜索筛选、按日分组、编辑删除）
- 离线暂存同步（含冲突处理、7天本地缓存）
- 深色模式（Light/Dark/System）
- 基础首页（精简版）

### v0.2 - 自动化记账（3 周）

- 通知监听自动记账 + 实时弹窗确认
- 短信解析（银行消费短信）
- 统计分析（趋势图 + 分类饼图 + 排行）
- 记账模板（保存常用、快捷复用）
- 智能分类学习（本地规则引擎）
- App Shortcuts（长按图标快捷入口）

### v0.3 - 预算与数据（2 周）

- 预算管理（总预算 + 分类预算 + 超支提醒）
- 消费洞察提醒（环比异常推送）
- 账单导入（微信/支付宝 CSV 文件解析）
- 周期记账（房租/工资等定期自动入库）
- 记账打卡激励（连续记账天数）

### v0.4 - 体验增强（2 周）

- 语音输入
- 桌面 Widget（快速记账 + 月度摘要）
- 通知栏快捷入口
- 应用锁（指纹/面部）
- 通知隐私模式
- 截图分享记账

### v0.5+ - 远期探索

- AI 对话查账
- 拍照 OCR 识别小票
- 屏幕 OCR 识别
- 悬浮球
- 地理位置 / 地图视图
- 多人共享账本（需后端支持）
